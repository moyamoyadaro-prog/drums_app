# ヒアリング・プロトコル（/gen-solo の Step 1）

`/gen-solo` は、いきなり既定値で埋めずに **対話式でヒアリング**してから組み立てる。
質問は `AskUserQuestion` で選択式に出し、**最後にオープンな自由記入**を一度受ける。
ここで得た回答をそのまま `BRIEF.md` の各項目へマッピングする。

## 原則

- ユーザーが最初の依頼文で既に答えている項目は **再質問しない**（埋まった項目は確認のみ）。
- 1回の `AskUserQuestion` は最大4問。下記を **3バッチ**に分けて聞く。
- 各選択肢は代表例。ユーザーは "Other" で自由入力できる（個人名・具体BPM など）。
- 全バッチ後に、平文で「他に指定・イメージはありますか？（自由記入）」を一度聞く＝オープン記入。

## バッチ1 — 曲とリズムの土台（どういう曲・どういうリズム）

| header | 質問 | 選択肢（例） | → BRIEF |
|---|---|---|---|
| ジャンル | スタイル/ジャンルは？ | Jazz / Funk・Fusion / Rock・Metal / Latin・Pop | `genre` `style/feel` |
| リズムの感じ | リズムの土台は？ | ストレート8分 / ストレート16分 / スウィング(3連) / シャッフル・ハーフタイム | `feel` |
| テンポ | テンポ感は？(BPM) | バラード(〜80) / ミドル(80〜120) / アップ(120〜180) / 速い(180〜) | `tempo` |
| タイム感 | ノリは？(BPMとは別) | ジャスト / 前ノリ(突っ込み気味) / 後ノリ(レイドバック) / おまかせ | `time_feel` |

## バッチ2 — 使い所と展開（どういう展開）

| header | 質問 | 選択肢（例） | → BRIEF |
|---|---|---|---|
| 使用シーン | どの場面で使う？ | 間奏ドラムソロ / トレード(4・8小節交換) / イントロ・フィル集 / エンディング | `scene` |
| 展開 | 全体の起伏は？ | 静→動の段階ビルド / 一定キープ / 緩急の波 / クライマックス一点 | `sections` `dynamics` |
| 長さ・難易度 | 規模と難易度は？ | 短め8小節・易 / 標準16小節・中 / 長め32小節+・難 / おまかせ | `bars` `difficulty` |
| 締め方 | 終わり方は？ | 大きいクラッシュで締め / decrescendoで静かに / 急停止(キメ) / おまかせ | `dynamics` |

## バッチ3 — 表現とキャラクター・制約

| header | 質問 | 選択肢（例） | → BRIEF |
|---|---|---|---|
| イメージドラマー | イメージするドラマーは？ | 特になし / あり（"Other"で名前を入力） | `reference_drummer` |
| 運指志向 | 手順(スティッキング)の志向は？ | こだわらない / リニア中心 / 伝統的(オルタネート・パラディドル) / ゴースト多用 | `sticking` |
| 避けたい要素 | 避けたい要素は？(複数可) | 特になし / ツーバス・速いキックは控えめ / タム回しは控えめ / 複雑な連符は避ける | `avoid` |

> **避けたい要素**は `multiSelect: true` で聞く（複数選択）。他は単一選択。
>
> **イメージドラマー**が「あり＋名前」のときは、その奏者の代表的な手癖（例: ビハインド・ザ・ビート、
> リニアフィル、3連の埋め方）を `notes` に短く言語化し、`phrase-library/` の `drummer:` タグでの
> 想起にも使う。名前は出力データには載せず brief 内メモに留める。
>
> **タイム感**は BPM ではなく置きどころ。譜面上は厳密に表現しきれないため、`notes` に意図を残し、
> 後ノリ/前ノリの傾向はゴーストやフレーズの食い方で近似する。
>
> **展開**と**使用シーン**の回答は、`SOLO-ARCHITECTURE.md` のアーク設計（提示→展開→クライマックス→着地）と
> 曲形式（12小節ブルース/AABA32/トレード4s…）へ写す。Step 1.6（構造パス）で `sections` を局面へ割り当て、
> モチーフ M とアーク・グリッドを決めてから音を置く。「静→動の段階ビルド」なら密度・強弱・音域の 3 曲線を
> 単調増加させ、頂点直前に“間”を置く。

## オープン記入（最後に必ず）

平文で1回聞く:

> 「他に取り込みたいフレーズ・参考曲・避けたいこと・自由なイメージがあれば教えてください（任意）」

ここで得たテキストは `BRIEF.md` の `notes` と `source` に反映する。

## マッピング後

埋まった `BRIEF.md` を**要約して提示**し、`sections` の小節合計＝`bars` が合うかだけ確認してから
Step 1.5（フレーズ想起）へ進む。
