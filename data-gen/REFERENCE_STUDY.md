# 既存ソロからの学習（reference study）

世の中の名ドラムソロから phrase-library を育てるための取り決め。実行は `/study-reference`
スキル。実行時 AI 不使用の不変則どおり、これは **PC の Claude Code が事前に行う**学習で、
アプリは結果（library → 生成ソロ → 備考）を読むだけ。

## 基本姿勢

- **音声/動画の自動採譜はしない**（未対応）。扱うのは **公開済みテキスト**の採譜・教則・解説情報のみ。
- 狙うのは原演奏の「丸写し」ではなく、**特徴的なフレーズ／デバイス（手癖）を短い断片**で取り込むこと。
  1エントリは原則 **1〜2小節**。これは精度・著作権の両面で安全側。
- **「忠実を狙う」≠「忠実だと断言する」**。裏取りできた範囲だけ高い `fidelity` を付け、
  できなければ格下げして正直に記録する。**精度を捏造しない**。

## fidelity（忠実度）の判定

| 値 | 条件 | 例 |
|---|---|---|
| `transcribed` | 複数の信頼できる採譜/教則テキストで、音価・スティッキング・楽器配分まで裏取り済み | 公開採譜が一致する定番フィル |
| `paraphrase` | 一般的な記述（「ここで速い三連フィル」等）からの再構成。具体音は近似 | 解説のみで音符は推定 |
| `idiom` | 特定ソロではなく、そのドラマー/ジャンルの一般的手癖 | 「Gadd 風リニア」全般 |

- 迷ったら**低い方**を選ぶ。`transcribed` は裏取りの URL（`refs`）必須。
- gen-solo が備考に出すのは原則 `transcribed` / `paraphrase`（=原曲が認知できる流用）。
  `idiom` は形が溶けるので、認知できる形で出したときのみ出典化する。

## 出典（refs）と帰属の扱い

- `refs` には裏取りに使った URL を入れる（採譜サイト・教則記事・解説など）。
- `drummer` / `song` は library エントリのタグと `<slug>.refs.json` に置く。
  生成 `.alphatex` 本体では **`\title` の「<ドラマー>風」表記のみ可**（fidelity と整合する控えめな帰属）。
  それ以外（譜面コメント・トラック名など notation 本体の他の場所）には**絶対に書かない**（CONVENTIONS.md §7）。
- 取り込むのは短い断片＋出典明記に留め、ソロ全体の複製はしない。

## 学習後の自動ソロ生成（毎回）

学習を回したら**毎回**、今回学んだフレーズ／イディオム**だけ**でソロを1本生成してアプリへ反映する
（生成→検証→commit/push まで自動。詳細は `/study-reference` スキル手順6〜7）。

- 使う素材は今回記録した library エントリに限定。**隙間埋めの無駄なオリジナルフレーズは作らない**
  （学習が浅いうちは反復・変位・休符で構成し、基礎語彙や創作で埋めない）。
- `\title` は「<ドラマー>風 …」。短く素朴なソロでも良い（語彙が育てば厚くなる）。

## Web 裏取りの進め方

1. 対象（ドラマー/曲/デバイス）で `WebSearch`。
2. 採譜・スティッキング・テンポ・楽器配分が読めるテキストを `WebFetch` で確認。
3. 複数ソースで一致する要素を `transcribed`、片側のみ/曖昧は `paraphrase` として alphaTex 化。
4. 出典 URL を `refs` に控える。

## library への記録

`phrase-library/README.md` の書式に従い、`source: reference:<artist>-<song>`、
`drummer` / `fidelity` / `refs` を埋めて該当ジャンルファイルへ追記する。
id は `<genre>-<scene>-<連番2桁>`。既存と重複・凡庸なものは登録しない。
